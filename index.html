Implementação

Descrição dos Nós

noh = "{"custo": 0,"quadra": {},"filhos": {},"pai": null,"heu": -6,"hsh": "\nbcda\n"}"
custo
inteiro, quantidade de elementos
quadra
Conjunto de Arrays
filhos
Conjunto de nós filhos
pai
link para o nó pai
heu
heurística do nó
hsh
identificador único

variáveis e códigos auxiliares

torre0 e torreF são variaveis globais e representam o estado inicial e final

nozes[]é um Array onde todo filho não repetido gerado fica armazenado. nozes[n] retorna o n-ésimo nó armazenado, n=0 é carregado durante a inicialização da página.

hashes{} é um conjunto dos identificadores dos nós. hashes.has(hsh) retorna verdadeiro se o hsh pertencer ao conjunto

geraFilhos(n) é uma função que recebe um nó n, lê o conjunto n.quadra que contêm as torres e permuta o topo de cada uma com todas as outras. Cada combinação é adicionada ao conjunto n.filhos

heuristica(q) é uma função que recebe um conjunto de Array e calcula a H2 do slide

Busca manual auxiliada

geraFilhos()

Alternadamente de cada torre destaca-se o topo, esta torre é chamada de pivô.

O topo é combinada com cada torre que não é pivô e com o chão.

A função recebe um nó e retorna um conjunto contendo todos os nós filhos possíveis

Larga

O filhos não repetidos são adicionados ao final de um Array

Em ordem crescente cada elemento do Array tem seus filhos gerados

Profunda

Os filhos não repetidos são adicionados ao final de um Array e de uma Pilha

O topo da pilha é retirados e gerado filhos

Gulosa

Ao invés de ser inserido no final da pilha, os filhos gerados são colocados dentro de seções com a mesma heurística

A Estrela

A diferença é que a inserção é ponderada pela soma da heurística com o custo

heurística

A laranja seria seria uma heurística real seguindo um caminho de aproximação. A verde uma admissível, que não subestima o valor real e vai a zero no estado final. A vermelha é a heurística utilizada.
